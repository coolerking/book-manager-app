# G02070 メッセージ一覧

## 1. 本書の位置付け

本書は「書籍管理Webアプリ」（以下、本システム）でユーザに表示する**メッセージ文言の一覧**を定義する。

各画面・各業務シナリオから参照されるメッセージを **メッセージID** で一意化し、文言・カテゴリ・表示方法・表示先画面を整合させる。

## 2. メッセージ運用ルール

[B01010] 5.5 / [G01010] 9章 を基本とし、本書で以下を確定する。

### 2.1 カテゴリ

| カテゴリ   | 表示形態         | 配色              | 表示時間              | アイコン |
| ---------- | ---------------- | ----------------- | --------------------- | -------- |
| 成功       | 通知バー（ヘッダ直下） | 緑系（color-success） | 3秒で自動クローズ     | ✓        |
| 警告       | 通知バー（ヘッダ直下） | 橙系（color-warning） | ユーザが閉じるまで残る | ⚠        |
| エラー     | 通知バー / 画面遷移先 | 赤系（color-error）   | ユーザが閉じるまで残る | ✕        |
| 入力エラー | フォーム欄直下   | 赤系（color-error）   | 再送信まで残る        | （アイコンなし） |
| 確認       | モーダルダイアログ | 既定スタイル        | ユーザが閉じるまで残る | （アイコンなし） |
| 情報       | 画面コンテンツ内に直接記述 | 既定スタイル | 該当画面表示中        | （アイコンなし） |

### 2.2 文言ルール

- **日本語のみ**で記述し、句点（`。`）で終える。
- 主語は省略する（「書籍を登録しました。」）。
- 数値は3桁区切りカンマを付ける（[B01010] 5.6）。
- 変数挿入は `{プレースホルダ名}` 形式で表現し、表示時にエスケープする。
- 技術的な詳細（スタックトレース等）は含めない（[B01010] 5.7）。
- 否定形より行動を促す表現を優先する（例: 「入力してください。」）。

### 2.3 メッセージID 採番ルール

- `MSG-{カテゴリ略号}-{連番3桁}` で採番する。
  - `S` = Success（成功）
  - `W` = Warning（警告）
  - `E` = Error（エラー、想定外含む）
  - `V` = Validation（入力エラー）
  - `C` = Confirm（確認）
  - `I` = Information（情報）
- 例: `MSG-S-001`、`MSG-V-002`

### 3.4 入力エラー（バリデーション）

入力エラーは項目単位で表示する（[B01010] 5.2 / [G01010] 6.2）。`{項目名}` は対象フォームの論理名（タイトル・著者・ISBN 等）が入る。`{n}` は閾値の数値。

| メッセージID | 文言                                              | 対象項目           | 種別                               |
| ------------ | ------------------------------------------------- | ------------------ | ---------------------------------- |
| MSG-V-001    | {項目名}は必須です。                              | title / author     | 必須欄が未入力                     |
| MSG-V-002    | {項目名}は{n}文字以内で入力してください。         | 全テキスト/メモ    | 桁数超過                           |
| MSG-V-003    | {項目名}は半角数字とハイフンで入力してください。  | isbn               | 形式不正                           |
| MSG-V-004    | {項目名}は0以上{n}以下の整数で入力してください。  | price              | 範囲外・非整数                     |
| MSG-V-005    | {項目名}は正しい日付（YYYY-MM-DD）で入力してください。 | purchase_date  | 不正な日付                         |
| MSG-V-006    | {項目名}に未来の日付は指定できません。            | purchase_date      | 未来日付（任意の業務ルール）       |

> **MSG-V-006 補足**: 「未来日付禁止」は本リリースでは**任意ルール**として実装する。サーバ側で許可するかは基本設計で確定する。

### 3.5 確認メッセージ（モーダル）

| メッセージID | 見出し               | 本文                                                             | 表示先 | トリガ                  | 関連UC |
| ------------ | -------------------- | ---------------------------------------------------------------- | ------ | ----------------------- | ------ |

## 4. 通知バーの表示位置・閉じる動作

[G01010] 9章 に従い、ヘッダ直下に通知バーを配置する。

| 項目             | 動作                                                                       |
| ---------------- | -------------------------------------------------------------------------- |
| 出現アニメーション | スライドダウン（上→下）。所要 200ms。                                     |
| 消滅アニメーション | フェードアウト。所要 200ms。                                              |
| 「×」ボタン      | 右端に配置。クリックで即時消滅。                                           |
| キーボード操作   | Tab で「×」ボタンへ移動可能。`Enter` または `Space` で消滅。               |
| 複数同時表示     | 1件のみ。新規メッセージが発生した場合は既存を置換する。                   |

## 5. 多言語対応

[B01010] 5.6 に従い、本リリースは**日本語のみ**とする。文言の i18n キー化は行わない。

将来 i18n 対応する場合は、本書の `MSG-XXX-NNN` 形式 ID をそのまま i18n キーに転用できる構造としている。

## 7. アクセシビリティ

- 通知バーには `role="status"`（成功/情報）または `role="alert"`（エラー/警告）を付与する。
- 「×」ボタンには `aria-label="メッセージを閉じる"` を付与する。
- 色のみで意味を伝えないため、各カテゴリのアイコン（✓ / ⚠ / ✕）を併用する（[B01010] 5.8 / [G01010] 10章）。

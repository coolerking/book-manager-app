# D03240 テーブル定義

本書は「書籍管理Webアプリ」（以下、本システム）の**物理データモデル**（SQLite 上のテーブル・列・型・制約・インデックス・DDL）を定義する。

## 2. 物理化方針

### 2.1 DBMS・ストレージ

| 項目             | 値                                                                   |
| ---------------- | -------------------------------------------------------------------- |
| DBMS             | SQLite 3（`better-sqlite3` v11 経由）                                |
| ファイル名       | `data/books.sqlite3`                                                 |
| ジャーナルモード | WAL（`PRAGMA journal_mode = WAL`）                                   |
| 同期モード       | `PRAGMA synchronous = NORMAL`（個人利用の応答性とのバランス）        |
| 外部キー         | `PRAGMA foreign_keys = ON`（将来の拡張に備えて常時 ON）              |
| 文字エンコーディング | UTF-8（SQLite 既定）                                              |
| タイムゾーン     | ローカル TZ。`datetime('now', 'localtime')` で取得                   |

### 2.2 命名規則

R03120 6 章 に従う。
- テーブル名: snake_case 複数形
- カラム名: snake_case 単数形
- インデックス名: `idx_{table}_{col1}_{col2}`
- ユニーク制約名（将来）: `uq_{table}_{col}`

### 2.3 型方針（SQLite の型親和性）

SQLite は動的型付け（型親和性）であるが、保守性のため**論理型 → SQLite 親和型**を以下のとおり統一する。

| 論理型           | SQLite 型宣言   | 補足                                                           |
| ---------------- | --------------- | -------------------------------------------------------------- |
| INTEGER          | `INTEGER`       | 64 bit 整数                                                    |
| STRING(n)        | `TEXT`          | 長さ制約は `CHECK (length(col) <= n)` で表現                   |
| DATE             | `TEXT`          | `YYYY-MM-DD` を文字列保存（[B01010] 5.2）                      |
| DATETIME         | `TEXT`          | `YYYY-MM-DD HH:mm:ss` を文字列保存                              |
| BOOLEAN（将来）  | `INTEGER`       | 0/1                                                             |

> SQLite には `VARCHAR` 等は存在せず `TEXT` に統一する。長さ制約は SQLite 側 `CHECK` で担保し、アプリ側のサーババリデーション（[A03130] 3.5）と二重で守る。

### 2.4 制約方針

| 種別               | 方針                                                                                                       |
| ------------------ | ---------------------------------------------------------------------------------------------------------- |
| 主キー             | サロゲートキー `id INTEGER PRIMARY KEY AUTOINCREMENT`                                                       |
| NOT NULL           | 必須列に NOT NULL を明示                                                                                   |
| 既定値             | `DEFAULT (datetime('now', 'localtime'))` を `created_at` / `updated_at` に付与                            |
| CHECK              | 長さ・範囲・形式を CHECK 制約で記述（DB 側の最終ガード）                                                  |
| 一意制約           | 本リリースでは設定しない（[D02020] 6 章：自然キー候補なし）                                                 |
| 外部キー           | 本リリースでは存在しない（エンティティ1個）                                                                 |
| トリガ             | `updated_at` の自動更新用 `BEFORE UPDATE` トリガを 1 個導入（5 章 DDL 参照）                                |

### 4.1 カラム一覧

| -- | -------------- | ---------- | ---------- | ---- | ------------------------------- | ----------------------------------------------------------------- | ------------------- |

| 6  | `purchase_date` | 購入日    | TEXT(DATE) | NULL | `NULL`                          | `purchase_date IS NULL OR purchase_date GLOB '____-__-__'`         | [D02020] 5.1.6      |

| 9  | `created_at`   | 登録日時   | TEXT(DT)   | NN   | `datetime('now', 'localtime')`  | `length(created_at) = 19`（`YYYY-MM-DD HH:mm:ss`）                  | [D02020] 5.1.9      |
| 10 | `updated_at`   | 更新日時   | TEXT(DT)   | NN   | `datetime('now', 'localtime')`  | `length(updated_at) = 19`                                          | [D02020] 5.1.10     |

> 凡例: `NN` = NOT NULL。

### 4.2 主キー / ユニーク

| 種別        | 構成                  | 補足                                                          |
| ----------- | --------------------- | ------------------------------------------------------------- |
| PRIMARY KEY | `id`                  | `AUTOINCREMENT` 採用。`sqlite_sequence` 経由で連番管理        |
| UNIQUE      | -                     | なし（[D02020] 6 章）                                          |

### 4.3 インデックス

| インデックス名                | 列                  | 種別        | 用途                                                                   |
| ----------------------------- | ------------------- | ----------- | ---------------------------------------------------------------------- |
| 主キーインデックス            | `id`                | PK 自動生成 | 1 件取得・更新・削除                                                   |
| `idx_books_created_at`        | `created_at DESC`   | 通常        | 既定ソート列（[B01010] 5.4）。一覧表示性能（NFR-B02）                  |
| `idx_books_title`             | `title`             | 通常        | 「タイトル」列ヘッダによる昇降ソート（[G02030] 3.3）                   |
| `idx_books_author`            | `author`            | 通常        | 「著者」ソート                                                         |
| `idx_books_publisher`         | `publisher`         | 通常        | 「出版社」ソート                                                       |
| `idx_books_purchase_date`     | `purchase_date`     | 通常        | 「購入日」ソート                                                       |
| `idx_books_price`             | `price`             | 通常        | 「価格」ソート                                                         |

> インデックスはソートの応答性（NFR-B02）担保のため、[G02030] 3.3 のソート可能列すべてに付与する。1,000〜数千件規模では領域オーバーヘッドは小（10 MB 未満想定、[D02020] 9 章）。

### 4.4 トリガ

| トリガ名                      | タイミング          | 内容                                                                  | 用途                              |
| ----------------------------- | ------------------- | --------------------------------------------------------------------- | --------------------------------- |
| `trg_books_set_updated_at`    | BEFORE UPDATE       | `NEW.updated_at = datetime('now', 'localtime')` を強制                | 更新日時の自動更新（[D02020] 10 章） |

> アプリ側でも `updated_at` をセットするが、DB トリガを最終防衛線とする。

## 5. DDL（正本）

`src/db/database.js` の起動時初期化で**1度きり**実行する。`IF NOT EXISTS` を付与しているため、再起動時に副作用を持たない。

```sql
-- ============================================================
--  書籍管理Webアプリ : スキーマ初期化 DDL
--  対象 DBMS  : SQLite 3 (better-sqlite3 v11)
--  対象ファイル: data/books.sqlite3
-- ============================================================

-- 0. PRAGMA（接続時に設定。スキーマには含まれないがメモとして併記）
-- PRAGMA journal_mode = WAL;
-- PRAGMA synchronous  = NORMAL;
-- PRAGMA foreign_keys = ON;

-- 1. テーブル
CREATE TABLE IF NOT EXISTS books (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,

    title         TEXT    NOT NULL
                  CHECK (length(trim(title))  BETWEEN 1 AND 200),

    author        TEXT    NOT NULL
                  CHECK (length(trim(author)) BETWEEN 1 AND 100),

    isbn          TEXT
                  CHECK (
                      isbn IS NULL
                   OR (length(isbn) BETWEEN 1 AND 17
                       AND isbn NOT GLOB '*[^0-9-]*')
                  ),

    publisher     TEXT
                  CHECK (publisher IS NULL OR length(publisher) <= 100),

    purchase_date TEXT
                  CHECK (
                      purchase_date IS NULL
                   OR purchase_date GLOB '____-__-__'
                  ),

    price         INTEGER
                  CHECK (price IS NULL OR (price BETWEEN 0 AND 9999999)),

    memo          TEXT
                  CHECK (memo IS NULL OR length(memo) <= 1000),

    created_at    TEXT    NOT NULL
                  DEFAULT (datetime('now', 'localtime'))
                  CHECK (length(created_at) = 19),

    updated_at    TEXT    NOT NULL
                  DEFAULT (datetime('now', 'localtime'))
                  CHECK (length(updated_at) = 19)
);

-- 2. インデックス
CREATE INDEX IF NOT EXISTS idx_books_created_at    ON books(created_at DESC);
CREATE INDEX IF NOT EXISTS idx_books_title         ON books(title);
CREATE INDEX IF NOT EXISTS idx_books_author        ON books(author);
CREATE INDEX IF NOT EXISTS idx_books_publisher     ON books(publisher);
CREATE INDEX IF NOT EXISTS idx_books_purchase_date ON books(purchase_date);
CREATE INDEX IF NOT EXISTS idx_books_price         ON books(price);

-- 3. トリガ : UPDATE 時に updated_at を自動更新
CREATE TRIGGER IF NOT EXISTS trg_books_set_updated_at
BEFORE UPDATE ON books
FOR EACH ROW
BEGIN
    UPDATE books
       SET updated_at = datetime('now', 'localtime')
     WHERE id = OLD.id;
END;
```

> 注意: SQLite の `BEFORE UPDATE` トリガから同じ行を `UPDATE` すると無限ループ的に見えるが、SQLite は再起しないため安全（`OLD.id` で同一行を 1 度だけ更新）。代替として `AFTER UPDATE` を採用してもよいが、本書では BEFORE で統一する。

## 6. CRUD と SQL テンプレート

[A03130] 3.3 のリポジトリインタフェースに対応する標準 SQL を以下に示す。リポジトリは原則これらをプリペアドステートメントとして保持する（NFR-E03）。

### 6.1 一覧（参照） `findAll`

```sql
-- :sort と :dir はホワイトリスト検証後に文字列連結（プリペアードに不可）
SELECT id, title, author, isbn, publisher, purchase_date, price, memo, created_at, updated_at
  FROM books
 ORDER BY :sort :dir
 LIMIT  :limit
 OFFSET :offset;
```

許容する `:sort` の値: `id` / `title` / `author` / `publisher` / `purchase_date` / `price` / `created_at`。
許容する `:dir` の値: `ASC` / `DESC`。それ以外はコントローラ側で 400 扱い（既定値にフォールバック）。

### 6.2 件数 `count`

```sql
SELECT COUNT(*) AS cnt FROM books;
```

### 6.3 1 件取得 `findById`

```sql
SELECT id, title, author, isbn, publisher, purchase_date, price, memo, created_at, updated_at
  FROM books
 WHERE id = :id;
```

### 6.4 登録 `create`

```sql
INSERT INTO books (title, author, isbn, publisher, purchase_date, price, memo)
VALUES (:title, :author, :isbn, :publisher, :purchase_date, :price, :memo);
```

> `id` / `created_at` / `updated_at` は省略（DB 既定値）。

### 6.5 更新 `update`

```sql
UPDATE books
   SET title          = :title,
       author         = :author,
       isbn           = :isbn,
       publisher      = :publisher,
       purchase_date  = :purchase_date,
       price          = :price,
       memo           = :memo
 WHERE id = :id;
```

`updated_at` はトリガが自動更新する。

### 6.6 削除 `remove`

```sql
DELETE FROM books WHERE id = :id;
```

## 10. 制約検証マトリクス

CHECK 制約とアプリバリデーション（[A03130] 3.5 / [G02070] 3.4）の対応。

| 列              | アプリバリデーション             | DB CHECK                                                         | 重複検証の意義                |
| --------------- | -------------------------------- | ---------------------------------------------------------------- | ----------------------------- |
| `title`         | 必須 / 1〜200                    | NOT NULL + 1〜200                                                | 仕様逸脱を最後にブロック      |
| `author`        | 必須 / 1〜100                    | NOT NULL + 1〜100                                                 | 同上                          |
| `isbn`          | 任意 / 0〜17 / 数字とハイフン    | NULL 許容 + 1〜17 + 数字ハイフンのみ                              | 同上                          |
| `publisher`     | 任意 / 0〜100                    | NULL 許容 + ≤ 100                                                | 同上                          |
| `purchase_date` | 任意 / `YYYY-MM-DD`              | NULL 許容 + `____-__-__` 形式                                     | 形式ガード                    |
| `price`         | 任意 / 0〜9,999,999 の整数        | NULL 許容 + 0〜9999999                                            | 範囲ガード                    |
| `memo`          | 任意 / 0〜1,000                  | NULL 許容 + ≤ 1000                                                | 同上                          |
| `created_at`    | アプリ非設定（DB 既定値）         | NOT NULL + 長さ 19                                                | DB 採番（[B01010] 5.9）        |
| `updated_at`    | アプリ非設定（DB トリガ）         | NOT NULL + 長さ 19                                                | DB 採番／トリガ更新           |

> `isbn` の CHECK 条件は SQLite の `GLOB` で「英字を含まない」を `NOT GLOB '*[^0-9-]*'` で表現する（空文字は事前にアプリ側で NULL に正規化する）。

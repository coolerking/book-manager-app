# B01030 システム化業務フロー

## 1. 本書の位置付け

本書は [B01020 システム化業務一覧](./B01020_システム化業務一覧.md) に列挙した4業務（B11010〜B11040）について、ユーザと本システム（書籍管理Webアプリ）の振舞いを業務フローとして示すものである。

記法・凡例・共通振舞いは [B01010 システム振舞い共通ルール](./B01010_システム振舞い共通ルール.md) の「3. 業務フローの記述ルール」と「5. 共通振舞いルール」に準拠する。

---

## 2. 凡例

- レーンは「ユーザ」「本システム」の2レーン構成（外部システムなし）。
- 開始 `((開始))` / 終了 `((終了))` / 判断 `{条件}`（分岐ラベルは「はい/いいえ」）。
- 画面名は `「画面名」` のように鉤括弧で囲む。
- 入力バリデーションは画面側・サーバ側の二重で行う（[B01010] 5.2）。

---

## 3. B11010 書籍を登録する

タイトル・著者を必須とし、ISBN・出版社・購入日・価格・メモは任意で入力する。

```mermaid
flowchart TD
    Start((開始)) --> A[ユーザ: 一覧画面で「新規登録」を押下]
    A --> B[システム: 「書籍登録画面」を表示]
    B --> C[ユーザ: 書籍情報を入力]
    C --> D[ユーザ: 「登録」ボタンを押下]
    D --> E{画面側バリデーション<br/>タイトル・著者は入力済か}
    E -- いいえ --> F[システム: エラーメッセージを表示]
    F --> C
    E -- はい --> G[システム: サーバ側バリデーション]
    G --> H{妥当か}
    H -- いいえ --> F
    H -- はい --> I[システム: 書籍データを保存]
    I --> J[システム: 「一覧画面」へ遷移し完了メッセージを表示]
    J --> End((終了))
```

## 4. B11020 書籍一覧を参照する

既定ソートは登録日時の降順、1ページ10件固定。0件時は登録画面への導線を表示する（[B01010] 5.4）。

```mermaid
flowchart TD
    Start((開始)) --> A[ユーザ: 「一覧画面」を開く<br/>またはページ番号/前後ボタンを押下]
    A --> B[システム: 該当ページのデータを取得<br/>10件/ページ・登録日時降順]
    B --> C{該当書籍が存在するか}
    C -- いいえ --> D[システム: 「登録された書籍はありません」と<br/>登録画面への導線を表示]
    D --> End((終了))
    C -- はい --> E[システム: 一覧表示・ページャ表示]
    E --> F{操作}
    F -- ページ移動 --> A
    F -- 列ヘッダクリック --> G[システム: 昇順/降順をトグルし再表示]
    G --> E
    F -- 終了 --> End
```

## 5. B11030 書籍を修正する

修正対象は1件、必須項目はタイトル・著者。

```mermaid
flowchart TD
    Start((開始)) --> A[ユーザ: 一覧で対象行の「修正」を押下]
    A --> B[システム: 対象書籍を取得し<br/>「書籍修正画面」に初期表示]
    B --> C[ユーザ: 値を編集]
    C --> D[ユーザ: 「更新」ボタンを押下]
    D --> E{画面側バリデーション<br/>タイトル・著者は入力済か}
    E -- いいえ --> F[システム: エラーメッセージを表示]
    F --> C
    E -- はい --> G[システム: サーバ側バリデーション]
    G --> H{妥当か}
    H -- いいえ --> F
    H -- はい --> I{対象書籍は存在するか}
    I -- いいえ --> J[システム: 「対象書籍が存在しません」を表示]
    J --> K[システム: 「一覧画面」へ遷移]
    K --> End((終了))
    I -- はい --> L[システム: 書籍データを更新]
    L --> M[システム: 「一覧画面」へ遷移し完了メッセージを表示]
    M --> End
```

## 6. B11040 書籍を削除する

確認ダイアログ必須、既定フォーカスは「キャンセル」、物理削除のみ（[B01010] 5.3）。

```mermaid
flowchart TD
    Start((開始)) --> A[ユーザ: 一覧で対象行の「削除」を押下]
    A --> B[システム: 確認ダイアログを表示<br/>既定フォーカス=キャンセル]
    B --> C{ユーザの選択}
    C -- キャンセル --> D[システム: ダイアログを閉じ一覧へ戻る]
    D --> End((終了))
    C -- 削除する --> E[システム: 対象書籍を取得]
    E --> F{対象書籍は存在するか}
    F -- いいえ --> G[システム: 「対象書籍が存在しません」を表示]
    G --> H[システム: 「一覧画面」を再表示]
    H --> End
    F -- はい --> I[システム: 書籍データを物理削除]
    I --> J[システム: 「一覧画面」へ戻り完了メッセージを表示]
    J --> End
```

## 7. 代表シーケンス（書籍登録）

画面とサーバの責務分担を補足する。記法は [B01010] 3.3 に準拠。

```mermaid
sequenceDiagram
    actor U as ユーザ
    participant V as ブラウザ(画面)
    participant S as サーバ(Node.js 24)
    participant D as データストア

    U->>V: 「書籍登録画面」を開く
    V-->>U: 入力フォームを表示
    U->>V: 書籍情報を入力し「登録」押下
    V->>V: 画面側バリデーション(必須:タイトル/著者)
    alt 失敗
        V-->>U: エラーメッセージ表示・最初のエラー欄へフォーカス
    else 成功
        V->>S: POST /books (書籍情報)
        S->>S: サーバ側バリデーション
        alt 失敗
            S-->>V: 400 + エラー詳細
            V-->>U: エラーメッセージ表示
        else 成功
            S->>D: INSERT 書籍レコード
            D-->>S: 登録結果(ID採番)
            S-->>V: 201 Created + 完了メッセージ
            V-->>U: 「一覧画面」へ遷移し完了通知
        end
    end
```

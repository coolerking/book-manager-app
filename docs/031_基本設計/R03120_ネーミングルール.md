# R03120 ネーミングルール

## 1. 本書の位置付け

本書は「書籍管理Webアプリ」（以下、本システム）の**ソフトウェア命名規約**を定義する。

[A03110 ソフトウェア論理構成] の4層モデルに沿って、各層で扱う識別子（ファイル名・モジュール名・関数名・変数名・DB識別子・URL・テンプレート変数・CSSクラス名 等）の命名規則を統一する。後続の S03210（クラス図）・P03210（APIルーティング仕様）・D03240（テーブル定義）・S04010 / P04010（実装）はすべて本書に従う。

前提とする上位ドキュメント:

- [B01010 システム振舞い共通ルール](../010_要件定義/B01010_システム振舞い共通ルール.md)
- [G01010 レイアウト共通ルール](../010_要件定義/G01010_レイアウト共通ルール.md)
- [A03110 ソフトウェア論理構成](./A03110_ソフトウェア論理構成.md)

## 2. 基本方針

| 観点               | 方針                                                                                                                            |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------- |
| 言語               | コード上のすべての識別子は**英語**（半角英数 + アンダースコア）。コメント・JSDoc・ログメッセージのみ日本語可。                  |
| 一貫性             | 「同じ概念には同じ語」「異なる概念には異なる語」。略語は本書 9章 のリストに登録したもののみ使用可。                              |
| 可読性             | 1〜2文字の変数名は禁止（ループカウンタ `i` を除く）。曖昧名（`data`, `info`, `tmp`, `obj` 等）はレビュー対象。                  |
| 安全性             | 予約語・既存ライブラリの export 名と衝突しないこと（例: `request`, `module` を変数名に使わない）。                              |
| ケースの統一       | レイヤごとに採用ケースを固定し、混在させない（3章のサマリ表参照）。                                                             |
| 文字エンコーディング | すべての識別子は ASCII 範囲のみで構成（NFR-F04 / [B01010] 5.9）。                                                              |

## 3. ケース規約サマリ

下表のケースを各種識別子のデフォルトとする。具体例は 4 章以降で示す。

| 対象                          | ケース                  | 例                                       |
| ----------------------------- | ----------------------- | ---------------------------------------- |
| JavaScript ファイル名         | camelCase               | `bookController.js`                      |
| JavaScript 変数・関数         | camelCase               | `findById`, `bookList`                   |
| JavaScript クラス・コンストラクタ | PascalCase             | `BookRepository`                         |
| JavaScript 定数（モジュールスコープ） | UPPER_SNAKE_CASE | `DEFAULT_PAGE_SIZE`                      |
| JavaScript ローカル定数       | camelCase               | `const errors = {}`                      |
| ディレクトリ名                | lowerSingular（短い場合 lower、長い場合 camelCase） | `routes/`, `controllers/`            |
| EJS テンプレート ファイル名    | snake_case + `.ejs`     | `book_list.ejs`, `book_form.ejs`         |
| EJS テンプレートに渡すデータキー | snake_case             | `book_list`, `page_count`                |
| URL パス                      | kebab-case（複数形名詞） | `/books`, `/books/:id/edit`             |
| HTML フォーム `name` 属性     | snake_case              | `purchase_date`, `price`                 |
| CSS クラス                    | kebab-case（BEM 風）    | `book-list__row--zebra`                  |
| HTML `id` 属性                | kebab-case              | `book-form`                              |
| DB テーブル名                 | snake_case（複数形）    | `books`                                  |
| DB カラム名                   | snake_case（単数形）    | `title`, `purchase_date`                 |
| 環境変数                      | UPPER_SNAKE_CASE        | `BOOK_APP_PORT`                          |
| 文書ID（メッセージID等）       | 既定（[G02070] 2.3）    | `MSG-S-001`, `NFR-E01`                   |

## 5. JavaScript（バックエンド）

### 5.1 ファイル名・モジュール分割

| 種別               | 形式                                | 例                                  |
| ------------------ | ----------------------------------- | ----------------------------------- |
| ルータ             | `{resource}.js`（複数形）           | `routes/books.js`                   |
| コントローラ       | `{resource}Controller.js`           | `controllers/bookController.js`     |
| リポジトリ         | `{resource}Repository.js`           | `repositories/bookRepository.js`    |
| インフラ           | 役割名 camelCase                    | `db/database.js`、`lib/logger.js`   |
| アプリ／サーバ     | `app.js`, `server.js`                | （固定名）                          |

- 1ファイル = 1責務（1モジュール）を原則とする。
- リソース名は **`book`（単数形）** を基準とし、URL／DB テーブル名のみ複数形（`books`）。

### 5.2 変数・定数・関数

| カテゴリ                 | ケース            | 規約                                                                                                |
| ------------------------ | ----------------- | --------------------------------------------------------------------------------------------------- |
| ローカル変数             | camelCase         | 動詞＋名詞は使わない。状態を表す名詞句で命名（例: `currentBook`, `errorMap`）。                    |
| 関数（関数式・宣言）     | camelCase         | 動詞＋目的語（例: `findById`, `buildErrorMap`）。Boolean を返す関数は `is`/`has`/`can` 接頭辞。   |
| クラス・コンストラクタ   | PascalCase        | 例: `class BookRepository {}` （将来クラス採用時）。                                                |
| モジュールスコープ定数   | UPPER_SNAKE_CASE  | 例: `const DEFAULT_PAGE_SIZE = 10;`、`const MAX_TITLE_LENGTH = 200;`                                |
| ローカル定数（変動しない値）| camelCase        | 一時的な計算結果や中間オブジェクト。                                                                |
| 内部用ヘルパ             | `_` 接頭辞は使わない | ESM/CJS の export 制御で隠蔽する。                                                                  |

#### 5.2.1 命名語彙の統一

| 概念                     | 採用する語        | 採用しない語                  |
| ------------------------ | ----------------- | ----------------------------- |
| 取得（単一）             | `findById`        | `getOne`, `read`, `select`    |
| 取得（一覧）             | `findAll`         | `list`, `getAll`, `select`    |
| 作成                     | `create`          | `insert`, `add`, `register`   |
| 更新                     | `update`          | `modify`, `edit`, `save`      |
| 削除                     | `remove`          | `delete`（JS 予約語っぽさ回避） |
| 件数取得                 | `count`           | `size`, `total`               |
| 整形                     | `format{Xxx}`     | `make`, `to`                  |
| 検証                     | `validate{Xxx}`   | `check`, `verify`             |

> リポジトリ／コントローラの関数名は本表に厳密に従う（S03210 / S04010 で再確認）。

### 5.3 ファイル先頭の `'use strict'` / ESM/CJS

- モジュール形式は **CommonJS（`require` / `module.exports`）** を採用する（NFR-C01 のシンプルさを優先）。
- ファイル先頭の `'use strict'` は記述不要（Node 24 で既定）。

### 5.4 引数オブジェクト規約

- 引数3個以上を取る関数は**オプションオブジェクト**にまとめる。

```js
// OK
repo.findAll({ page: 1, pageSize: 10, sort: 'created_at', dir: 'desc' });

// NG（位置引数が多すぎる）
repo.findAll(1, 10, 'created_at', 'desc');
```

- オプションオブジェクトのキー名は snake_case（DB カラム名に揃える、5.5 参照）か camelCase かを **モジュール単位で統一**する。リポジトリ層は **snake_case**（DB カラム名と一致）、コントローラ層は **camelCase** とする。境界（コントローラ → リポジトリ）でケース変換する。

### 5.5 DB ↔ JS のキー変換ポリシー

| 場所                    | キーの形式                | 例                                  |
| ----------------------- | ------------------------- | ----------------------------------- |
| DB カラム               | snake_case                | `purchase_date`                     |
| リポジトリの戻り値      | snake_case（DB に揃える） | `{ id, title, purchase_date, ... }` |
| コントローラ → リポジトリ の入力 | snake_case          | リポジトリ呼出時は DB 形式に統一    |
| ビューに渡すモデル      | snake_case を継承         | EJS テンプレートで `<%= book.purchase_date %>` |
| HTML フォーム `name`    | snake_case                | `<input name="purchase_date">`      |
| URL クエリパラメタ      | snake_case                | `?sort=created_at&dir=desc`         |

> 例外: コントローラ内のローカル変数（一時オブジェクト）は camelCase で命名してもよい。境界では明示的に変換する。

## 6. データベース（SQLite）

| 種別                  | 規約                                                                                                                                  | 例                                              |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| テーブル名            | snake_case 複数形                                                                                                                     | `books`                                         |
| カラム名              | snake_case 単数形。論理ER（[D02020]）の属性名に揃える。                                                                              | `title`, `purchase_date`, `created_at`          |
| 主キー                | `id` 固定                                                                                                                             | `id INTEGER PRIMARY KEY AUTOINCREMENT`          |
| タイムスタンプ        | `{動詞}_at` 形式                                                                                                                      | `created_at`, `updated_at`                      |
| 外部キー（将来）      | `{参照テーブル単数形}_id`                                                                                                              | `author_id`                                     |

| 件数カラム（将来）    | `{名詞}_count`                                                                                                                         | `comment_count`                                 |
| インデックス名        | `idx_{table}_{col1}_{col2}`                                                                                                            | `idx_books_created_at`                          |
| ユニーク制約名（将来）| `uq_{table}_{col}`                                                                                                                     | `uq_books_isbn`                                 |

> 詳細スキーマと DDL は [D03240 テーブル定義] で確定する。

## 7. HTTP / URL / フォーム

### 7.1 URL パス

- 基本 RESTful。リソース名は **複数形・kebab-case**（書籍のみのため kebab を要する単語は登場しない）。
- 識別子は `:id` で表現（数値、サーバ採番）。
- 詳細は [P03210 APIルーティング仕様] で確定。

| 操作                    | パス                              |
| ----------------------- | --------------------------------- |
| 一覧（参照）            | `GET /books`                      |
| 登録フォーム表示        | `GET /books/new`                  |
| 登録実行                | `POST /books`                     |
| 編集フォーム表示        | `GET /books/:id/edit`             |
| 更新実行                | `POST /books/:id`                 |
| 削除実行                | `POST /books/:id/delete`          |

### 7.2 クエリパラメタ

| パラメタ名     | 意味                            | 例                                  |
| -------------- | ------------------------------- | ----------------------------------- |
| `page`         | ページ番号（1始まり）           | `?page=3`                           |
| `sort`         | ソート列（DB カラム名）         | `?sort=created_at`                  |
| `dir`          | ソート方向（`asc` / `desc`）    | `?dir=desc`                         |
| `flash`        | フラッシュメッセージキー        | `?flash=created` ※ [G02070] 6章    |

### 7.3 HTML フォーム

| 項目                    | 規約                                                                                       |
| ----------------------- | ------------------------------------------------------------------------------------------ |
| `name` 属性             | DB カラム名と一致（snake_case）                                                            |
| `id` 属性               | `book-form-{name}` 形式（kebab-case）。例: `book-form-title`                              |
| label の `for` 属性     | input の `id` と一致                                                                       |
| エラー表示要素 `id`     | `error-{name}`（例: `error-title`）                                                        |
| ボタン `name`           | 動詞（`submit`, `cancel`）。送信用ボタンは name 属性を省略してもよい                       |

### 7.4 リクエスト/レスポンスヘッダ

| 項目                | 値                                                          |
| ------------------- | ----------------------------------------------------------- |
| `Content-Type` 受信 | `application/x-www-form-urlencoded; charset=UTF-8`          |
| `Content-Type` 送信 | `text/html; charset=UTF-8` / `text/css; charset=UTF-8`      |
| 文字コード          | UTF-8 固定                                                  |

## 9. 略語・用語の取り扱い

略語は**本表に登録したもののみ**コード中で使用可。それ以外は完全形で記述する。

| 略語   | 完全形              | 使用箇所                   |
| ------ | ------------------- | -------------------------- |
| `id`   | identifier          | DB / 変数 / URL            |
| `db`   | database            | モジュール名のみ           |
| `ctrl` | controller          | 使用しない（完全形 `controller` を採用） |
| `repo` | repository          | 使用しない（完全形 `repository` を採用） |
| `req` / `res` / `next` | Express ハンドラ引数  | Express ハンドラ内のみ |
| `env`  | environment         | 環境変数関連のみ           |
| `pkg`  | package             | 使用しない                 |
| `ISBN` | International Standard Book Number | DB カラム `isbn`（小文字）／表示時は大文字 |
| `URL`  | -                   | コメント・ドキュメントのみ。コード上は `url` |
| `HTTP` | -                   | 同上                       |

> 「short ≠ 良い名前」。長くなっても完全形を優先する。

## 10. ファイル先頭・コメント・ログ

### 10.1 JS ファイル先頭

各 JS ファイルの先頭に、1行 JSDoc コメントで責務を1行記述する（[A03110] 5章 の役割と整合）。

```js
/** リポジトリ層: books テーブルに対する CRUD を提供する。 */
```

### 10.2 ログ文言

- ログは**日本語可**。ただしカテゴリ識別子は ASCII 固定。
- フォーマット: `[YYYY-MM-DD HH:mm:ss] [LEVEL] [module] メッセージ`
  - LEVEL: `INFO` / `WARN` / `ERROR`（[A01010] NFR-C02）
  - module: `app` / `route` / `controller` / `repository` / `db` のいずれか
- 個人情報を含む可能性のあるフィールド値はログに出さない（書籍タイトル等は可、自由記述メモは ERROR 時のみ）。
